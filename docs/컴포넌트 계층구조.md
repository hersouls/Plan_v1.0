# Moonwave Plan 컴포넌트 계층구조 (Component Hierarchy)

## 개요

이 문서는 Moonwave Plan 프로젝트의 컴포넌트 계층구조를 정의합니다. 모든 컴포넌트는 디자인가이드.md의 스타일 가이드라인을 100% 준수하며, 재사용성과 일관성을 보장하는 구조로 설계되었습니다.

## 1. 컨테이너 (Container) 계층

### 1.1 Layout Container

```
📁 Layout Container (src/layout/)
├── Header.tsx - 메인 헤더 컴포넌트

## 5. 컴포넌트 상속 관계

### 5.1 기본 상속 구조

```
📁 Base Components (src/components/ui/)
├── GlassCard.tsx ← 모든 카드의 기본 스타일
├── button.tsx ← 모든 버튼의 기본 스타일

├── input.tsx ← 모든 입력 필드의 기본 스타일

## 10. 컴포넌트 사용 가이드라인

### 10.1 컴포넌트 선택 기준

1. **기본 UI 컴포넌트**: 공통적인 UI 요소 (버튼, 카드, 배지 등)
2. **특화 컴포넌트**: 특정 기능에 최적화된 컴포넌트
3. **페이지 컴포넌트**: 전체 페이지 레이아웃
4. **컨테이너 컴포넌트**: 여러 컴포넌트를 감싸는 레이아웃

### 10.2 스타일 적용 우선순위

1. Design Tokens (최우선)
2. UI Component 기본 스타일
3. Specialized Component 스타일
4. Page-specific 스타일 (최후)

### 10.3 컴포넌트 확장 시 고려사항

- 기존 UI 컴포넌트 상속 우선
- 디자인 토큰 사용 필수
- 재사용성 고려
- 접근성 준수

## 11. 파일 구조 요약

```
src/components/
├── ui/ - 기본 UI 컴포넌트
├── task/ - 태스크 관련 컴포넌트
├── family/ - 가족 관련 컴포넌트
├── dashboard/ - 대시보드 컴포넌트
├── settings/ - 설정 관련 컴포넌트
├── ai/ - AI 관련 컴포넌트
├── charts/ - 차트 컴포넌트
├── common/ - 공통 컴포넌트
└── layout/ - 레이아웃 컴포넌트
```

이 계층구조는 Moonwave Plan의 확장성과 유지보수성을 보장하며, 일관된 사용자 경험을 제공합니다.
